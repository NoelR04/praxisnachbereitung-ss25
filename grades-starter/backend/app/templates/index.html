🎓 Grades Starter

Demo-App mit FastAPI, Postgres und MQTT für die Domäne Studierende / Module / Grade.

Dieses Projekt ist das Referenzprojekt für Tag 4. Die eigentliche Übung entwickelt ihr in eurer Inventar-App aus Tag 3 weiter.

Tag 4 erweitert den Stand aus Tag 3 um zwei zusätzliche Aspekte: Alembic-Migrationen und einen klaren Report-/Export-Baustein für Excel.

Migrationen

Dieses Projekt erzeugt das Schema bewusst nicht mehr über db/init, sondern über Alembic:

docker compose exec api alembic upgrade head

Mit persistentem Postgres-Volume bleiben Daten auch nach docker compose down erhalten. Erst docker compose down -v setzt die DB wirklich zurück.

MQTT: Neue Note per Topic anlegen

Der Service hört auf das Topic grades/new. Payload (JSON) z. B.:

{ "student_id": 1, "module_id": 2, "grade_value": "1,7" }
